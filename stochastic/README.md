# NTT-friendly обратимые MWC-генераторы и NTT-hash-функции

> Быстрый GPU-ориентированный PRNG / Hash с обратимостью шага, NTT-ускорением и физически-информированным дизайном, подобным SPONGE-SQUEEZE

## 1. Введение в MWC-генераторы и их свойства

Проект исследует MWC (Multiply-With-Carry) как основу для модульного аналога CRC-хэша — линейного, обратимого, с естественным avalanche-эффектом смешивания за счет выбора модуля.

На простых числах Прота ($p = k·2^s + 1$) MWC становится NTT-friendly (метод ускорения матричного умножения): итерацию можно интерпретировать, как полиномиальное умножение в кольце, диагонализовать матрицы перестановок через NTT и ускорить до $O(n log n)$.

Это перекликается с принципами построения FHE/ZK-friendly хешей (Poseidon, Rescue, Pasta и др.), где вместо битовых операций — модульная арифметика в кольце полиномов.

**Multiply-With-Carry (MWC)** — линейный генератор, обобщающий LCG, и аналог CRC/LFSR в кольце ℤ/pℤ, где p — большое простое число (часто Mersenne, pseudo-Mersenne, Proth или Rizel вида $p = k·2^s ± 1$).

Основное преимущество шага MWC — *обратимость* и *avalanche effect*: наличие как forward-шага `next()`, так и backward-шага `prev()`, а также эффективная реализация пропуска шагов и слияния сегментов `jump()` / `merge()`. Это превращает генератор в полноценную арифметику над счётчиком итераций и открывает возможности параллельной генерации. 

Цель проекта — создать *быстрый, современный GPU-ориентированный PRNG / хэш-примитив*  с хорошими статистическими свойствами, потенциалом NTT-ускорения матричных операций и возможностью добавления физически-информированного базиса для синтеза широкого класса хэш-функций (port-Hamiltonian / Dirac structure + Ring-LWE шум).

## 2. MWC128 — базовая конструкция и PRNG-hash

MWC128 использует 128-битное состояние и работает по схеме:

$x ← (A · x) mod P$, где $P = A · 2^{64} − 1$ — большое простое число (числа Ризеля, но могут применяться и числа Прота).

Типичный хэш-вариант использования MWC-итерации в составе Хэш-функции (PRNG_hash.md):

```c
uint64_t mwc128_hash(const uint8_t* data, uint64_t len, uint128_t seed) {
    uint128_t h = unmix(seed);
    for (int i = 0; i < len >> 3; i++) {
        h += *(uint64_t*)data; data += 8;
        h = (uint64_t)h * A1 + (h >> 64);  // MWC-итерация
    }
    // обработка остатка + финальный mix
    return mix(h);
}
```

**Выбор множителя A1** — ключевой: A1 должен быть генератором мультипликативной группы в ℤ/2⁶⁴ℤ, обеспечивать хороший avalanche и равномерность заполнения - это дополнительные требования.

**Миксеры** (MXM, MUM, MWC, RRX, RNG, MRC):
- MXM — классика Murmur/SplitMix (xor-shift + multiply)
- MUM — multiply → xor/add reduction
- RRX — rotate-xor (циркулянтные матрицы над GF(2))
- MRC — CRT-подход с двумя модулями

**Векторизация и folding** — позволяют параллельный хэш на AVX-512 / NEON, lazy reduction, со слиянием хеш-сегментов.

**Тестирование** — SMHasher (avalanche, BIC, collisions, speed), TestU01, custom SAC / difficulty. MWC128 проходит все 188 тестов SMHasher3.

## 3. Обратимые MWC-генераторы (Reversible RNG)

Обратимость достигается через *Shoup-модульное умножение* на константу (без переполнения) и использование A как обратного значения к сдвигу $2^s$.

Пример MWC32 reverse (из ReversibleRNG.md):

```c
uint32_t MWC32_rev(uint32_t s, uint32_t A) {
    const uint32_t p = (A << 16) - 1u;
    const uint64_t w = 0x1p48 / p;          // предвычисленный \bar{w}
    uint64_t q = (s * w) >> 32;
    uint64_t r = ((s - q * A) << 16) + q;
    return (r < p) ? r : r - p;
}
```

Аналогично для MWC64/128 — обратное умножение на $A$ (или деление на $2^s$).

**Аналогии**:
- LCG: x_{k-1} = a^{-1} (x_k - c) mod p
- PCG: ядро обратимо, scrambler должен быть биективным
- XorShift: обратная матрица (инверсия сдвигов и обратимая бинарная матрица трансформации)

**Применение**:
- adjoint Monte-Carlo / обратные симуляции
- undo/redo в играх и симуляциях
- stochastic rounding в низкоточностных нейросетях
- постквантовые схемы (комбинация с Ring-LWE шумом)

## 4. NTT-friendly дизайн и связь с числами Прота

**Выбор модуля** — $p = k · 2^s + 1$ (Proth prime) с примитивным корнем порядка $2^s$ → позволяет использовать полный NTT размером $2^s$.

**Связь с циклотомическими полиномами** $Φ_{2^{s+1}}(X) ≈ X^{2^s} + 1$ → диагонализация циклических сдвигов и умножений на X в NTT-базисе.

**Гибридный раунд**:
- MWC-step (линейный carry + multiply)
- NTT-mix (быстрое полиномиальное умножение / NTT в кольце)
- Аффинные (обратимые) преобразования двоичных векторов на входе и выходе

**Преимущества**:
- O(n log n) на микс вместо O(n²)
- jump/skip через возведение в степень в NTT-домене
- естественная интеграция с hardware-ускорителями NTT (FHE, lattice crypto)

## 5. Конструкция sponge / PRNG-хэш на обратимом MWC + NTT

- **Состояние**: 256–1024+ бит (несколько 128-битных MWC-слов)
- **Absorb**: MWC-step + добавление данных + NTT-mix
- **Squeeze**: несколько раундов чистого миксера (без ввода)
- **Обратимость** используется для: предсказания орбит, merge сегментов, undo операций, adjoint-вычислений
- **Нелинейность** (опционально): квантовый шум, простые ARX-слои, нелинейная H(x) в Dirac-стиле

## 6. Оценка качества и спектральные свойства

**Lattice test / спектральный тест** — индикатор качества линейных генераторов (MWC, LCG, XorShift).

Параметры спектрального теста LCG:
- $d_s$ — минимальное расстояние между гиперплоскостями в s-мерном пространстве
- $F_s = d_s · m^{1/s}$ (чем выше — тем лучше, идеал > 0.8–1.2)

Проблема LCG: низкие $F_s$ в малых размерностях → точки выборки данных лежат на регулярной решётке → предсказуемость. Это не проявляется явно для MWC (конгруэнтный генератор по модулю простого числа) при степени 64 и выше. Генераторы проходят статистические тесты TestU01 включая Furier, LineaComp и Спектральные тесты Кнута. Однако, проявляется периодическая структура следования паттернов в тесте "Trump Aces" и наличие двух непересекающихся орбит, одна из которых исключает степени двойки "тузы".

**Рекомендации по усилению**:
- Увеличение состояния и числа раундов
- Weyl-increments / добавление квантового шума (Ring-LWE стиль)
- Нелинейные слои (модульное сложение, RRX миксеры и MDS-перестановки)
- Dirac-структура, как индуцированный базис (кососимметричная J + диссипация R)

## 7. Перспективы: физическая интерпретация и Ring-LWE

- Dirac-Lattice DRNG / хэш с обучаемыми параметрами
- Гибридные конструкции MWC + ARX + NTT для пост-вантовой криптографии
- RAG-поиск по сжатым данным, ZK-доказательства, CRT-MRC векторный хэш
  
Стохастическая модель строится на базе *Dirac structure + NTT-диагонализация матрицы перестановок + квантовый шум*:

Это позволяет:
- O(n log n) шаг генерации
- шум затрудняет спектральный анализ для атаки
- градиентное обучение параметров (J, R, σ) модели

Связь с FHE (CKKS/BFV) — прямая: NTT как ключевая оптимизация, шум как основа безопасности. Построение векторного хеш с использованием CRT-MRC. RAG поиск по сжатым данным. Доказательства с нулевым разглашением. 

> PHS и Dirac structure возникают в контексте пост-квантовой нейронной сети — это не физическая модель, а математический класс (структурный инвариант).  
> Матрицы с эрмитовым сопряжением, гамильтонианы и порождаемые гамильтоновы матрицы для многомерных обобщенных переменных, кососимметричные операторы — это голая математика. Физика может быть любой — в любой геометрии, размерности, со своей мерой, — но основа остаётся неизменной: основа отражает гильбертово (многомерное) пространство нормируемых собственных функций, которое интерпретируется, как вероятностное пространство.  
> Структура Дирака в PHS фиксирует сохранение мощности (внутренней нормы) и стабильность генерации — тот же структурный принцип, который сохраняется после замены операций.
> Именно этот класс определяет элемент квантовой нейронной сети с подменой операций:  
элемент сети, в частности, может быть IIR-фильтром (ARMA-генератором последовательности), многомерным базисным полиномом Бернштейна, как базис вероятностного пространства, в концепции KSN/KAN, или матричным элементом подобным PHS.
> Квантовый шум используется, как математический механизм фундаментального принципа наименьшего действия (stochastic rounding, как механизм диссипации и сохранения информации).
