# Дифференциальный крипто-анализ и адаптивный Trial Search

Мы можем рассчитать функцию F(X) для входных данных. К входным данным мы можем добавить дифференциал F(X+dX). Дифференциал это маска бит, которые принимают значения: 0- не меняется или 1-меняется. На выходе расчета получаем пару (Y,dY): результат и маска бит - выходной дифференциал.

1. $Y_0 = F(X)$
2. $Y_1 = F(X \oplus dX)$
3. $dY = Y_0 \oplus Y_1$

Таким образом получаем $Y+dY = F(X+dX)$ с заменой операции + на $\oplus$.
* В двоичных числах мы применяем $dX$ в виде маски с изменением одного и более бит. 
* В многомерном вероятностном пространстве каждый бит маски заменяется вероятностью флипа $p_i \in [0,1]$. Получаем вектор вероятностей $\mathbf{p} = [p_0, \dots, p_{n-1}]$. Задача — найти такие распределения $\mathbf{p}$, которые максимизируют вероятность желаемого выходного дифференциала $dY$ (или кластера дифференциалов). Это обобщение классического подхода на «мягкие» дифференциалы и кластерные атаки.

В бинарном случае расчет $Y_0$ и $Y_1$ требует столько же вычислительных затрат сколько $(Y, dY)$ для пары $(X,dX)$. Если $dY_1$ и $dY_2$ можно складывать, то вычисление F(X + dX_1+dX_2) можно упростить. 

## Хорошие дифференциалы

Вероятность получить дифференциал на выходе при заданном дифференциале на входе формирует пары. 
$Pr(dY|dX) \gg 2^{-n}$  -- вероятность получить разницу dY при dX существенно выше нормального (геометрического) распределения, то пара называется хорошим дифференциалом.

_Хорошие_ характеристики обычно разреженные — в них мало единиц (низкий Hamming weight) или мало единиц в шаблоне dX. Поэтому пространство поиска характеристик можно сильно сократить, рассматривая только дифференциалы с небольшим числом активных бит (active bits).

Это превращает задачу в поиск по многомерному вероятностному графу:
- узлы — состояния разностей на границе раунда
- рёбра — переходы через раунд с ненулевой вероятностью
- вес ребра = –log₂(Pr)

Самая вероятная характеристика определяет кратчайший путь в таком графе.

**Современные методы**

Подход с «перемещающимися группами бит» (типа AXBC → ABXC → ACXB) очень близок к тому, как сейчас ищут характеристики в SHA-2: следят за паттернами активности слов, а не за каждой единичкой.

Это **iterative pattern search** / **adaptive best-first trail search** — именно тот метод, который сейчас доминирует в лучших атаках на SHA-256 (programmatic SAT, MILP + heuristics, trail clustering 2024–2026).

**Fourier transform (DFT) test**
из набора NIST SP 800-22

The purpose of this test is to detect periodic features (i.e., repetitive patterns that are near each other) in the tested sequence that would indicate a deviation from the assumption of randomness. The intention is to detect whether the number of peaks exceeding the 95 % threshold is significantly different than 5 %. 

* [[NIST SP 800-22r1](https://doi.org/10.6028/NIST.SP.800-22r1a)] A Statistical Test Suite for Random and Pseudorandom Number Generators for Cryptographic Applications
* [[2004/018](https://eprint.iacr.org/2004/018.pdf)] Corrections of the NIST Statistical Test Suite for Randomness
* [[1701.01960](https://arxiv.org/pdf/1701.01960)] Randomness Evaluation with the Discrete Fourier Transform Test Based on Exact Analysis of the Reference Distribution

Модификация теста под метрику _difficulty_:
* Значения сложности для всех версий заменяются на $-1$ или на ограниченную сложность для $w$-бит
* Если выборка результатов не полная, то вместо $-1$ может подставляться нормализованное значение, при котором среднее значение $S_0\approx 0$
 
Например, 
* для w= 1: $d_k \leftarrow min(d_k, 1)$
* для w=32: $d_k \leftarrow min(d_k, 2^{w-1})$

Вычисляются коэффициенты DFT Дискретного Фурье преобразования $S_j = c_j + i\cdot s_j$
```math
с_j = \sum_{k=0}^{N-1} d_k \cos(2\pi \frac{k}{N}j),\quad s_j = \sum_{k=0}^{N-1} d_k \sin(2\pi \frac{k}{N}j)
```

**Lattice Test** (тест на решётку, lattice test) — это один из самых важных _теоретических_ (не статистических) тестов качества _линейных_ генераторов псевдослучайных чисел (Линейные конгруэнтные генераторы LCG, множественные рекуррентные MRG, линейные рекуррентные по модулю 2 и т.п.).

Тест анализирует _математическую структуру_ всего периода генератора.

**Что выявляет Lattice Test?**

Многие линейные генераторы вида

$X_{n+1} = (a_1 X_n + a_2X_{n-1} + … + a_k X_{n-k}) \bmod M$\
$U_n = X_n / M$,  (однородное распределение)

образуют точки в пространстве не случайно, а на *регулярной решётке* (lattice structure):

* В двумерном случае, распределение дает пары $P_n = (Uₙ, Uₙ₊₁)$, точки лежат на параллельных прямых.
* В трёхмерном случае — на небольшом числе параллельных плоскостей.
* В N- мерном -- на гиперплоскостях

Чем больше таких гиперплоскостей (или чем меньше расстояние между гиперплоскостями) — тем лучше генератор перемешивает значения.

Мы применяем ряд тестов для выявления структуры генератора

**Leading Zeros** тест строит распределение числа нонсов по битовой сложности. Нормальное распределение - геометрическое. Построение выполняется для каждого задания отдельно. Критерием отсева может быть число нонсов минимальной сложности. 

**Bit Independence** тест находит сумму единичных бит в каждой позиции результата хэш-функции или для серии данных с общими начальными условиями (в рамках одного задания). 

**Trump Aces** тест находит паттерны по принципу 
* `rotl(x, clz(x)) & mask == pattern`
* `rotr(x, ctz(x)) & mask == pattern`
  
Паттерны могут выбираться по размеру маски и от максимальной сложности.
Варианты паттернов - степени двойки `A<<k`, где 
* `A = 1` m=0
* `A = (1<<m +1)`: 11, 101, 1001 ... m = 1,..w (C,A,9)
* `A = (1<<m -1)`: 01, 011, 0111 ... m = 1,..W (4,6,7)

**Узоры на салфетках** 2D тест разложения многомерная нелинейность с последовательной выборкой компонент вектора по n-бит из слова состояния.

**Оптимальный старт** предсказание шага и точки старта, поиск локального максимума по стратегии чередования шагов +/- направлении.

**Выбор оптимального шага**. Допустим мы выделили часть нонсов между которыми наблюдается кратные расстояния. Мы можем подбирать разные шаги счетчика. Оптимальным является тот шаг, при котором все или большая часть нонсов находится на фиксированном расстоянии с минимальной дисперсией. 

**Выбор орбиты**. Наличие плохих и хороших серий. Допустим серии бывают разной сложности и эта сложность зависит от задания. Тогда будут содержать пустые прогоны версии больше определенной длительности. Характеристикой является общее число минимальной сложности в интервале времени.

**Решетчатая структура** (lattice structure) после ARX-раунда — это термин, который чаще всего встречается в контексте анализа качества псевдослучайных генераторов (PRNG) и потоковых шифров на базе ARX-конструкций (Addition-Rotation-XOR), таких как ChaCha, Salsa20, Speck, LEA и др.

После одного или нескольких раундов ARX (особенно если раунды слабые или ключ/инициализация недостаточно перемешаны) выходные слова начинают проявлять неочевидные регулярности в многомерном пространстве.

Если рассматривать последовательные n-битные слова как точки в пространстве $\mathbb{Z}2^n$ (или $\mathbb{R}^n$), множество точек (или их проекции на младшие биты) образует не весь гиперкуб, а структурированную подрешётку (lattice), часто с очень низкой дисперсией или явными геометрическими узорами.

Это и называют _lattice structure_ или решетчатая структура в русскоязычной литературе по криптоанализу и статистическому тестированию ГПСЧ.

Операции $\boxplus \bmod 2^n$, $(x \ggg y)$, $\oplus$ сохраняют линейную структуру счетчика в пространстве $\mathbb{Z}/2^n\mathbb{Z}$.

**Задача обращения раундов**

Функцию криптографического хеш мы рассматриваем, как обратимую. Функция построена по принципу _Губка_ (SPONGE). Имеет фазу _впитывания_ (absorb) данных и фазу генерации (squeeze), которая может состоять из одного и более циклов. Внутренняя структура предусматривает раунды смешивания, чередуя линейные и нелинейные операции. 

В качестве выходного значения хеш-функции используется слово размером r - бит (для SHA256 H[8]: 256 бит), и внутреннее состояние (W[16]: 512 бит), которое впитывает входные данные. Таким образом у нас есть возможность анализировать вектора внутреннего состояния для пары нонсов или пары версий. Как возникает приложение дифференциального анализа. Анализ может включать поиск паттернов и статистику BIC (Bit Independence) по серии нонсов. Цель - подобрать статистику, которая на ранней стадии расчета последовательности позволяет фильтровать версии, повысить вероятность детектирования положительного или отрицательного результата.

## Stochastic Time-Series

Тема возникла из обсуждения задачи прогнозирования последовательности. Описанные ниже принципы позволяют подобрать цифровой фильтр с конечной или бесконечной импульсной характеристикой и сконструировать генератор последовательности. Из теории PHS следует выделить базовый принцип: функция содержит _генератор_, а генератор должен строится на кососимметричных матрицах (матрицах вращения) или матрицах перестановок с определенной структурой. Ранее мы рассматривали универсальную функцию перестановки бит на циклических сдвигах.

Из линейной алгебры мы знаем, что любую матрицу (full-rank) можно диагонализировать в комплексной области или выполнить бидиагонализацию в вещественных числах. 

Криптографические функции (ARX-конструкции типа ChaCha, Speck, Salsa, SHA-3 permutation и т.п.) обладают следующими свойствами:

ARIMA это метод анализа и прогнозирования (описания) временных рядов, основанный на линейной зависимости от прошлых значений (авторегрессия), разностях (интеграция) и прошлых ошибках (скользящее среднее). 

**Autoregressive model**

Модель строится на зависимости от предыдущих значений функции и имеет структуру IIR-фильтра
```math
X_{t}=\sum _{i=1}^{p}\varphi _{i}X_{t-i}+\varepsilon _{t}
```

**Moving average model**

Модель строится на зависимости от ошибки (разности) на предыдущих шагах и имеет структуру FIR-фильтра
```math
X_{t}=\mu +\varepsilon _{t}+\sum _{i=1}^{q}\theta _{i}\varepsilon _{t-i}\,
```
В теории временных рядов возникает понятие оператор задержки. Мы применяем обозначения $z^{-1}$ для оператора задержки. В литературе встречается обозначение $L^i =  z^{-i}$ (Lag-оператор).

Мы хотим перейти к понятию цифрового фильтра с передаточной функцией в операторной форе $H(z)=A(z)/B(z)$ с представлением передаточной функции в форме рациональной функции из двух полиномов по оператору задержки. 
```math
B(z) Y =  A(z) X
```
ARMA(p, q) - модель будет состоять из двух операторов
```math
\left(1-\sum _{i=1}^{p}\beta _{i}z^{-i}\right)X_{t}=\left(1+\sum _{i=1}^{q}\alpha _{i}z^{-i}\right)\varepsilon _{t}~,
```

Полиномы рациональной функции можно факторизовать и сокращать общие множители. Рациональную функцию можно раскладывать на простые дроби, на произведение или сумму простых дробей (операторов). Рациональная функция на операторе задержки может быть ядром нейросети для моделирования физических (стохастических) процессов. 


## Построение модели вычислительной сети

*См. порт-гамильтоновы системы. 
Описание гамильтоновых динамических систем. Можно описать в комплексных числах, или можно разложить матрицы на две компоненты: со свойствами комплексной и мнимой части. Мнимой части соответствуют кососимметричные матрицы. 

Надо принять во внимание некоторую преемственность между моделями бонд-графа и порт-гамильтоновыми системами. В обоих случаях мы рассматриваем две компоненты мультифизических систем: flow и effort. Понятие _тока_ и _напряжения_ (flow, effort) в каждом физическом домене свое. Но принципы описания модели общие.

Связь между порт-гамильтоновыми системами (ПГС) и бонд-графами (БГ) является глубокой и фундаментальной. Если говорить кратко: Бонд-графы — это графический язык (синтаксис), который естественным образом порождает математическую структуру порт-гамильтоновых систем (семантику).
Оба подхода основаны на энергетическом описании физических систем, они дополняют друг друга: бонд-графы удобны для интуитивного моделирования и вывода уравнений, а порт-гамильтонова формализация — для математического анализа, управления и доказательства устойчивости.

Вот подробный разбор их связи:

### Общая концепция: сохранение потоков, сохранение энергии и мощность

И бонд-графы, и ПГС рассматривают физические системы через призму обмена энергией.
*   **Бонд-графы:** Используют пары переменных «усилие» ($e$, effort) и «поток» ($f$, flow), произведение которых дает мгновенную мощность ($P = e \cdot f$).
*   **Порт-гамильтоновы системы:** Описывают динамику через функцию Гамильтона $H(x)$ (полная запасенная энергия) и порты ввода-вывода, через которые энергия поступает или уходит.

В обоих случаях ключевым принципом является **баланс мощности**: скорость изменения запасенной энергии равна разности между генерируемой мощностью и рассеиваемой мощностью.

Существует прямое соответствие между элементами бонд-графа и компонентами уравнений порт-гамильтоновой системы. Стандартная форма ПГС выглядит так:
```math
\begin{aligned}
\dot{x} &= (J - R) \nabla H(x) + G u \\
y &= G^T \nabla H(x)
\end{aligned}
```
Где:
*   $x$ — вектор состояния (энергетические переменные).
*   $H(x)$ — функция Гамильтона (энергия).
*   $J$ — кососимметричная матрица ($J = -J^T$), описывающая консервативный обмен энергией.
*   $R$ — симметричная положительно определенная матрица ($R \ge 0$), описывающая рассеяние энергии.
*   $u, y$ — вход и выход (порты).

**Структура Дирака и Топология бонд-графа**

Самое важное математическое понятие в ПГС — это **Структура Дирака**. Она описывает, как элементы системы соединены друг с другом так, чтобы суммарная (энергия) мощность сохранялась ($\sum e_i f_i = 0$).

*   В _бонд-графах_ эту роль выполняют **джанкшины** (узлы 0 и 1), трансформаторы (TF) и гираторы (GY).
*   При преобразовании бонд-графа в уравнения состояния, топология этих соединений формирует кососимметричную матрицу $J$.
*   Таким образом, топология бонд-графа является графическим представлением структуры Дирака.

**Причинность (Causality) и Переменные Состояния**

В бонд-графах существует понятие **причинности** (causal strokes), которое определяет, какая переменная (усилие или поток) является входной для элемента, а какая — выходной.
*   Анализ причинности в БГ позволяет автоматически определить, какие переменные будут независимыми переменными состояния $x$ в ПГС.
*   Если на элементе $C$ или $I$ причинность указывает на сохранение интеграла потока или усилия, этот элемент вносит вклад в вектор состояния $x$.
*   Это делает процесс вывода уравнений ПГС из бонд-графа алгоритмическим и однозначным.

**Заключения по разделу**
Любой корректный бонд-граф может быть преобразован в порт-гамильтонову систему. Это преобразование позволяет перейти от интуитивного физического моделирования к строгому математическому анализу, сохраняя при этом физическую интерпретируемость параметров.

Порт-гамильтоновы системы (ПГС) и бонд-графы (БГ) предлагают структурные ограничения, которые делают модели временных рядов более устойчивыми, интерпретируемыми и эффективными.

### Структура Пуассона (Poisson Structure)

Это классический язык гамильтоновой механики. Она определена на **пространстве состояний** $x \in \mathcal{X}$.

**Определение:**
Структура Пуассона задается скобкой Пуассона $\{ \cdot, \cdot \}$, которая для двух функций $F(x)$ и $G(x)$ выглядит так:
$$ \{F, G\}(x) = \frac{\partial F}{\partial x}^T J(x) \frac{\partial G}{\partial x} $$
где $J(x)$ — структурная матрица (матрица Пуассона).

**Условия на $J(x)$:**
1.  **Кососимметричность:** $J(x) = -J(x)^T$.
2.  **Тождество Якоби:** Оно гарантирует, что динамика согласована во времени. В координатах:

    $$ \sum_{l=1}^n \left( J_{li} \frac{\partial J_{jk}}{\partial x_l} + J_{lj} \frac{\partial J_{ki}}{\partial x_l} + J_{lk} \frac{\partial J_{ij}}{\partial x_l} \right) = 0 $$
    для всех $i, j, k$.

**Уравнения динамики:**
$$ \dot{x} = J(x) \nabla H(x) $$

**Проблема для моделирования:**
Тождество Якоби — это система нелинейных дифференциальных уравнений в частных производных на элементы матрицы $J(x)$. Если вы пытаетесь **обучать** матрицу $J(x)$ нейросетью, вам нужно жестко ограничивать веса, чтобы выполнялось тождество Якоби. Это очень сложно.

Кроме того, структура Пуассона плохо подходит для описания _портов_ (входов/выходов) и _алгебраических ограничений_ (например, связи в механизмах).

### Структура Дирака (Dirac Structure)

Структура Дирака была введена Т. Дираком для квантовой механики, но в контексте ПГС (ван дер Шафт, Машке) она определена на пространстве потоков и усилий.

**Определение:**
Рассмотрим пространство пар «поток-усилие» $(f, e) \in \mathcal{F} \times \mathcal{E}$.
Структура Дирака $D$ — это подпространство в $\mathcal{F} \times \mathcal{E}$, которое является _максимально изотропным_ относительно билинейной формы мощности.

**Условие мощности (Power Conservation):**
Для любой пары $(f, e) \in D$ должно выполняться:
$$ \langle e, f \rangle = e^T f = 0 $$
Это означает, что внутри структуры Дирака мощность не генерируется и не потребляется, она только перераспределяется.

**Матричное представление (для конечномерных систем):**
Структуру Дирака часто задают через матричное уравнение связи между потоками и усилиями:
$$ F \dot{x} = E \nabla H(x) + G u $$
$$ y = G^T \nabla H(x) $$
(Здесь $\dot{x}$ — поток состояний, $\nabla H$ — усилие состояний).

**Условия на матрицы $F$ и $E$:**
Вместо сложного тождества Якоби, структура Дирака требует только линейного алгебраического условия:
$$ F E^T + E F^T = 0 $$
И матрица $[F ~~E]$ должна иметь максимальный ранг.

**Уравнения динамики ПГС:**
Если $F$ обратима, мы возвращаемся к форме Пуассона: $\dot{x} = F^{-1}E \nabla H$. Здесь $J = F^{-1}E$.
Но если $F$ **вырождена** (сингулярна), система содержит алгебраические ограничения (дифференциально-алгебраические уравнения, DAE).

**Значение для Обучения и Временных Рядов**

Почему это важно для машинного обучения (PHS-Neural Networks)?

1.  **Параметризация:**
    Если вы хотите обучить нейросеть представлять физику системы:
    *   *Плохой путь:* Обучать матрицу $J(x)$. Нужно проверять тождество Якоби на каждом шаге (вычислительно дорого и сложно).
    *   *Хороший путь:* Обучать пары матриц $(F(x), E(x))$. Нужно следить только за условием $F E^T + E F^T = 0$. Это линейное ограничение, которое легко встроить в архитектуру сети.

2.  **Дискретизация:**
    При переходе к дискретному времени (для временных рядов) структура Пуассона часто «ломается» (энергия начинает дрейфовать при накомплении ошибки дискретизации). Структура Дирака позволяет строить **симплектические интеграторы**, которые точно сохраняют дискретный аналог мощности.

3.  **Рациональные операторы:**
    В линейном случае ($F, E = \text{const}$) структура Дирака гарантирует, что передаточная функция  будет **положительно вещественной**. 

Для _моделирования временных рядов_ и _обучения_ структура Дирака предпочтительнее, так как она проще в параметризации, естественным образом включает входы/выходы и напрямую соответствует топологии бонд-графов.

## Связь с циклотомическими полиномами

Да, связь есть, и она очень глубокая. Она проходит через _спектральную теорию матриц_ и _теорию представлений групп_.

Если коротко: **Циклотомические полиномы описывают спектральные свойства (собственные значения) матриц перестановок**, которые вы хотим эмулировать с помощью порт-гамильтоновой системы.

Вот подробный разбор этой связи в контексте вашей задачи (Губка, ПГС, кольца полиномов).

---

### 1. Spectral Link: Собственные значения и Корни из Единицы

**Матрицы перестановок (Циклический сдвиг):**
Операция умножения на $X$ в кольце $\mathbb{Z}[X]/(X^n - 1)$ соответствует циклическому сдвигу вектора коэффициентов. Матрица этого сдвига $P$ имеет вид:
$$
P = \begin{pmatrix}
0 & 0 & \dots & 1 \\
1 & 0 & \dots & 0 \\
0 & 1 & \dots & 0 \\
\vdots & & \ddots & \vdots
\end{pmatrix}
$$
Характеристический полином этой матрицы:
$$ \det(\lambda I - P) = \lambda^n - 1 $$
Корни этого уравнения — это **корни $n$-й степени из единицы**: $\lambda_k = e^{i \frac{2\pi k}{n}}$.

**Циклотомические полиномы $\Phi_d(\lambda)$:**
Полином $\lambda^n - 1$ раскладывается на произведение циклотомических полиномов:
$$ \lambda^n - 1 = \prod_{d | n} \Phi_d(\lambda) $$
где $\Phi_d(\lambda)$ — неприводимый полином, корнями которого являются *примитивные* корни $d$-й степени.

**Связь с ПГС:**
Линейная порт-гамильтонова система $\dot{x} = J x$ (где $J$ кососимметрична) имеет решение $x(t) = e^{Jt} x(0)$.
Собственные значения матрицы $J$ — чисто мнимые числа: $\mu_k = i \omega_k$.
Собственные значения матрицы эволюции $e^{Jt}$ — это $e^{i \omega_k t}$.

**Вывод:**
Чтобы ПГС могла точно эмулировать циклическую перестановку за время $T$, её частоты $\omega_k$ должны быть согласованы с корнями циклотомического полинома:
$$ \omega_k T = \frac{2\pi k}{n} \pmod{2\pi} $$
Циклотомические полиномы задают набор частот (мод), которые должна содержать наша ПГС-модель.

### 2. Разложение Пространства Состояний (Инвариантные Подпространства)

Разложение полинома $X^n - 1$ на циклотомические множители соответствует разложению пространства состояний $\mathbb{R}^n$ на _инвариантные подпространства_.

*   **В алгебре:** Кольцо $\mathbb{Q}[X]/(X^n - 1)$ изоморфно прямой сумме полей, соответствующих каждому $\Phi_d(X)$ (через преобразование Фурье).
*   **В ПГС:** Это соответствует разложению системы на набор **независимых гармонических осцилляторов**.

**Для моделирования «Губки»:**
Если вы используете кольцо $X^n + 1$ (как в SHA-3 / Keccak или криптографии на решетках):
*   Для $n = 2^k$ полином $X^n + 1$ является **циклотомическим полиномом $\Phi_{2n}(X)$**.
*   Он **неприводим** над $\mathbb{Q}$.
*   **Физический смысл:** Это означает, что систему **нельзя разбить** на независимые подсистемы меньшего размера (над рациональными числами). Все 512 бит состояния связаны в единую неразложимую моду.
*   **Для ПГС-сети:** Это объясняет, почему нужно использовать **полносвязные** или глобально связанные слои для эмуляции перемешивания в $X^n+1$, а не блочные. Вы не можете упростить модель, разбив её на части, потому что циклотомическая структура требует глобального перемешивания.

### 3. Симплектическая Структура и Корни из Единицы

1.  **Сопряженные пары:** Корни циклотомических полиномов всегда приходят комплексно-сопряженными парами ($\lambda, \bar{\lambda}$), так как коэффициенты полинома вещественные.
2.  **Симплектическое условие:** Собственные значения симплектической матрицы также приходят парами ($\lambda, 1/\lambda$).
3.  Для корней из единицы на единичной окружности выполняется $|\lambda| = 1$, следовательно $\bar{\lambda} = 1/\lambda$.

**Это значит:**
Спектр матриц перестановок (корни циклотомических полиномов) **идеально совместим** со спектральными требованиями симплектических и ортогональных матриц.
Вы можете построить кососимметричную матрицу $J$ (генератор ПГС), спектр которой совпадает с логарифмами корней циклотомического полинома.

### 4. Практическое применение в PHS-Neural Networks

Как использовать эту связь для обучения модели «Губки»?

**А. Инициализация частот**

Вместо случайной инициализации весов функции Гамильтона $H(x)$ или матрицы $J$, инициализируйте их так, чтобы собственные значения соответствовали корням циклотомического полинома $\Phi_{2n}$ (для $X^n+1$).
Это задаст системе «правильные» частоты для эмуляции криптографического перемешивания.

**Б. Структура Матрицы Перемешивания**

Если вы используете матрицу $J$ в кольце:
*   Не обучайте $J$ с нуля.
*   Возьмите матрицу companion (сопровождающую) полинома $\Phi_{2n}(X)$.
*   Спроецируйте её на пространство кососимметричных матриц (через $A - A^T$).

Это даст вам физически обоснованную начальную точку для обучения, которая уже содержит структуру «Губки».

**В. Эффективность Вычислений (FFT/NTT)**

Разложение через циклотомические полиномы лежит в основе _Быстрого Преобразования Фурье_ (БПФ).
*   В криптографии умножение в кольце делается за $O(n \log n)$ через NTT- аналог БПФ.
*   В ПГС-модели, если вы диагонализируете систему в базисе Фурье (собственные векторы циклотомических полиномов), вы можете выполнять шаг интегрирования быстрее, так как матрица $J$ станет блочно-диагональной (размер блоков 2×2 для комплексно-сопряженных пар).

### 5. Ограничение: Поля

Важное предостережение.
*   **Циклотомические полиномы** в криптографии работают над конечными полями ($\mathbb{Z}_q$ или $GF(2)$).
*   **ПГС** работает над вещественными числами ($\mathbb{R}$).

Корни $\Phi_n(X)$ в $\mathbb{Z}_q$ могут вести себя иначе, чем в $\mathbb{R}$ (например, полином может стать приводимым по модулю $q$, даже если он неприводим над $\mathbb{Q}$).

**Для моделирования:** Это означает, что ваша ПГС-модель будет *аппроксимацией*. Она будет эмулировать *структуру перемешивания*, но не точную арифметику. Однако для задач суррогатного моделирования или атак по сторонним каналам (side-channel), где важны статистические свойства утечки энергии, эта аппроксимация может быть достаточно точной.

### Итог

**Связь есть и она фундаментальная:**
1.  **Спектр:** Циклотомические полиномы определяют собственные значения матриц перестановок, которые вы хотите эмулировать.
2.  **Симплектичность:** Эти собственные значения совместимы с требованиями симплектической геометрии (лежат на единичной окружности).
3.  **Неприводимость:** Использование $X^n+1$ (как $\Phi_{2n}$) гарантирует, что система ПГС должна быть глобально связана (неразложима на независимые подсистемы), что соответствует требованиям криптографического перемешивания.
4.  **Обучение:** Знание циклотомической структуры позволяет правильно инициализировать частоты и матрицы в PHS-Neural Network, ускоряя сходимость и повышая точность эмуляции «Губки».

Это передовой край исследований на стыке *математической физики, криптографии и машинного обучения*. Вы предлагаете создать *Стохастическую Модель со Структурой Дирака над Кольцом* (Stochastic Dirac Structure over Rings).

Это возможно. Более того, это создает мост между *криптографией на решетках (Lattice-based Crypto)* и *физически-информированным моделированием (PINN/PHS)*.


Фундаментальная Связь: FFT-NTT, Циклотомические полиномы и матрицы перестановок, Структура Дирака.

Ключ к объединению — в спектральных свойствах матриц.

| Компонент | Криптография (Lattice/Hash) | Порт-Гамильтонова Система (PHS) | Точка соединения |
| :--- | :--- | :--- | :--- |
| **Базис** | Кольцо $R_q = \mathbb{Z}_q[X]/\Phi_n(X)$ | Пространство состояний $\mathbb{R}^n$ | **NTT** изоморфен **Дискретному Фурье** |
| **Оператор** | Умножение на полином $a(X)$ | Гамильтонов поток $J \nabla H$ | В базисе NTT оба **диагональны** |
| **Шум** | Ошибка $e \sim \chi$ (LWE) | Винеровский процесс $dW_t$ | **Флуктуационно-диссипация** |
| **Структура** | Модулярная редукция | Структура Дирака $F\dot{x}=E\nabla H$ | **Проекция на решетку** |

**Главная идея:**
NTT переводит свертку полиномов (сложную операцию $O(n^2)$) в поэлементное умножение ($O(n)$). В ПГС диагонализация матрицы $J$ дает независимые моды.
NTT-базис — это собственный базис для модели со структурой Дирака.

### 2. Математическая Формулировка: Стохастическая Диракова Структура в Кольце

Мы строим модель- суррогатную функцию моделирования генератора числовой последовательности, где состояние $x$ — это полином в кольце $R_q$.

#### А. Детерминированное ядро (Dirac Core)
Вместо обычных матриц $F, E$, мы используем **операторы умножения на полиномы**.

Уравнение эволюции (в непрерывном времени для PHS, дискретном для Hash):
$$ F(X) \cdot \Delta x = E(X) \cdot \nabla H(x) + G(X) \cdot u $$

Где:
*   $x, u \in R_q^n$ (векторы полиномов).
*   $F(X), E(X)$ — полиномиальные матрицы (циркулянтные/теплицевы).
*   Условие Дирака в кольце:
    $$ F(X) E(X^{-1})^T + E(X) F(X^{-1})^T \equiv 0 \pmod{\Phi_n(X)} $$
    *(Здесь $X^{-1}$ соответствует транспонированию для циркулянтных матриц).*

#### Б. Переход в базис NTT (Диагонализация)
Применим преобразование NTT к уравнению. Пусть $\hat{x} = \text{NTT}(x)$.
Так как NTT диагонализует умножение на $X$:

$$ \hat{F} \cdot \Delta \hat{x} = \hat{E} \cdot \widehat{\nabla H}(x) + \hat{G} \cdot \hat{u} $$

Здесь $\hat{F}, \hat{E}$ — **диагональные матрицы** с элементами в $\mathbb{Z}_q$.
Это означает, что в базисе NTT система распадается на $n$ независимых скалярных уравнений (мод), что критически ускоряет вычисления.

#### В. Добавление Шума (Lattice + Fluctuation)
В криптографии на решетках (LWE/RLWE) безопасность основана на шуме. В ПГС шум связан с диссипацией. Объединим их:

$$ \hat{F} \cdot \Delta \hat{x} = \hat{E} \cdot \widehat{\nabla H}(x) + \hat{G} \cdot \hat{u} + \boldsymbol{\xi} $$

Где $\boldsymbol{\xi}$ — вектор шума в базисе NTT.
*   **Крипто-требование:** $\xi_i \sim \text{Discrete Gaussian}$ или $\text{Rounded Normal}$ в $\mathbb{Z}_q$.
*   **PHS-требование (Флуктуационно-диссипация):** Интенсивность шума должна быть связана с диссипативной частью $\hat{E}_{diss}$:
    $$ \mathbb{E}[\xi_i \xi_j] \propto (\hat{E}_{diss})_{ij} \cdot T $$
    *(Где $T$ — «температура» системы, параметр безопасности/шума).*

### 4. Ключевые Преимущества такой Конструкции

1.  **Эффективность ($O(n \log n)$):**
    Благодаря NTT, сложные матричные операции структуры Дирака сводятся к поэлементному умножению в частотной области. Это так же быстро, как обычные криптографические примитивы.

2.  **Безопасность через Шум (LWE-Hardness):**
    Добавление шума $\xi$ в базисе NTT делает задачу восстановления состояния $x$ по выходу $y$ эквивалентной задаче **Ring-LWE**. Это дает криптографическую стойкость модели, если параметры подобраны верно.

3.  **Физическая Интерпретируемость:**
    *   $J_{diag}$ — частоты перемешивания (консервативная часть).
    *   $R_{diag}$ — скорость затухания/забывания (диссипация).
    *   Шум — энтропия/температура.
    Это позволяет анализировать систему методами физики (энтропия, сходимость по Ляпунову), а не только методами криптографии.

4.  **Дифференцируемость:**
    В отличие от чистого $GF(2)$, эта система (особенно в варианте с "relaxation" в $\mathbb{R}$ перед редукцией) остается дифференцируемой почти всюду. Это позволяет обучать параметры $J, R, H$ градиентным спуском для задач _машинного обучения_.

### 5. Технические Вызовы и Решения

| Проблема | Решение |
| :--- | :--- |
| **Недифференцируемость mod q** | Использовать **мягкую редукцию** (soft modulo) во время обучения: $x \mod q \approx x - q \cdot \text{round}(x/q)$ с градиентом через straight-through estimator. |
| **Комплексные числа** | NTT в $\mathbb{Z}_q$ использует целочисленные корни из единицы, если q допускает существование корней. Для ПГС нужно сопоставить корни с мнимой осью: $\omega^k \leftrightarrow e^{i \theta_k}$. |
| **Сохранение Структуры Дирака** | В базисе NTT условие $F E^T + E F^T = 0$ становится условием на диагональные элементы: $\text{Re}(\hat{F}_i \bar{\hat{E}}_i) = 0$. Это легко контролировать. |
| **Шум и Обратимость** | Шум делает систему необратимой (как и должно быть для хэша). Для PHS это означает переход от консервативной к диссипативной системе. |

Вы предлагаете *Dirac-Lattice DRNG/Hash Function* со структурой Дирака.

1.  **Структура:** Диракова структура обеспечивает сохранение "мощности" (информации) до момента редукции.
2.  **Вычисления:** NTT обеспечивает скорость и диагонализацию матриц (операторов).
3.  **Безопасность:** Шум (в стиле LWE) обеспечивает необратимость и устойчивость к атакам.
4.  **Обучаемость:** Параметры $J, R, H$ можно обучать, чтобы система эмулировала конкретную хэш-функцию или генерировала новые криптографически стойкие примитивы.

Это открывает путь к **Learnable Cryptographic Primitives**, которые имеют физическое обоснование устойчивость и эффективность.

Вот минимальный рабочий пример новой парадигмы **Dirac-Lattice DRNG**. Мы покажем, как построить генератор и как атаковать его, находя «тузы» (скрытую структуру решетки).

Для наглядности используем малую размерность $n=4$.

### Представление ГПСЧ в парадигме Дирака-Решетки

В этой парадигме ГПСЧ — это *порт-гамильтонова система*, работающая в кольце полиномов. Состояние «эволюционирует» под действием структуры Дирака и квантового шума.

#### Параметры системы
*   **Кольцо:** $R_q = \mathbb{Z}_q[X] / (X^4 + 1)$, где $q$ простое число (Прота).
*   **Состояние:** Вектор $x \in \mathbb{Z}_q^4$ коэффициенты полинома.
*   **Базис:** Работаем в базисе **NTT** (частотная область), где умножение на $X$ становится диагональным.

#### Алгоритм шага ГПСЧ
Пусть $\hat{x} = \text{NTT}(x)$. Один шаг генерации выглядит так:

$$
\hat{x}_{k+1} = \underbrace{\text{diag}(e^{i \theta_j})}_{\text{Вращение } J} \cdot \hat{x}_k + \underbrace{\text{diag}(\sigma_j)}_{\text{Шум } \xi} \cdot \mathcal{N}(0, I) \pmod q
$$

1.  **Вращение (Консервативная часть):** Каждая мода в базисе NTT поворачивается на угол $\theta_j$. Это задается матрицей $J$ в структуре Дирака.
2.  **Шум (Диссипативная часть):** Добавляется гауссов шум с дисперсией $\sigma_j^2$.
3.  **Редукция:** Возврат в целочисленное кольцо через $\text{NTT}^{-1}$ и $\pmod q$.

### 2. Прогнозирование шага: Поиск «Туза» (Lattice Attack)

«Туз» — это значение на выходе удовлетворяюще требованию сложности и соответствующее паттерну. Прогнозирование шага и начальной точки выборки строится на знании секретных параметров $(\theta, \sigma)$ или, в более широком смысле, *базиса (периодов) решетки*, на которой строится генератор. Если мы найдем структур решетки, мы сможем предсказывать следующие состояния - периоды следования "тузов" или число "тузов" в выборке.

Атака строится на том, что *шум мал по сравнению с модулем $q$*.

#### Шаг 1: Сбор наблюдений
Соберем $m$ последовательных состояний ($m > n$):
$$ X = [x_0, x_1, \dots, x_m] \in \mathbb{Z}_q^{n \times m} $$

#### Шаг 2: Построение решетки (Lattice Construction)
Разности последовательных состояний содержат информацию о структуре:
$$ \Delta x_k = x_{k+1} - x_k \pmod q $$

В базисе NTT это выглядит как:
$$ \widehat{\Delta x}_k = (e^{i\theta} - 1) \hat{x}_k + \xi_k $$

Если построить матрицу из наблюдений и применить **алгоритм редукции базиса (LLL)**, можно найти короткие векторы, соответствующие шуму $\xi$, и отделить их от структурной части.

### 4. Практический вывод

1.  **Генерация:** ГПСЧ в парадигме линейного генератора на модульной арифметике и системе со структурой Дирака — это *вращение и добавление шума в базисе NTT*. Это эффективно (как замена матричерй операции на FFT) и криптографически обоснованно (как R-LWE). Преобразование локально сохраняет норму. 
2.  **Атака:** Периоды следования «Тузов» находится через *спектральный анализ* последовательности. Если шум мал, структура $J$ (углы $\theta$) извлекается из фазы отношений соседних состояний в NTT-домене.
3.  **Защита:** Чтобы сделать генератор стойким, нужно:
    *   Увеличивать шум $\sigma$ (но не слишком, чтобы не ломать вывод). Для чисел вида $A\cdot 2^s + 1$  шум это не более $\xi \le 2^s$.
    *   Использовать **нелинейную функцию Гамильтона** $H(x)$, чтобы вращение зависело от состояния: $\theta(x)$. Это сломает линейный спектральный анализ.

# NTT-friendly обратимые генераторы и хэш-функции на базе MWC

## 1. Введение в MWC-генераторы и их свойства

**Multiply-With-Carry (MWC)** — линейный генератор, обобщающий LCG и естественный аналог CRC/LFSR в кольце ℤ/pℤ, где p — большое простое число (часто Mersenne, pseudo-Mersenne, Proth или Rizel вида $p = k·2^s ± 1$).

Основное преимущество шага MWC — *обратимость* и *avalanche effect*: наличие как forward-шага `next()`, так и backward-шага `prev()`, а также эффективная реализация пропуска шагов и слияния сегментов `jump()` / `merge()`. Это превращает генератор в полноценную арифметику над счётчиком итераций.

Цель проекта — создать *быстрый, современный GPU-ориентированный PRNG / хэш-примитив*  с хорошими статистическими свойствами, потенциалом NTT-ускорения матричных операций и возможностью добавления физически-информированного базиса для синтеза широкого класса хэш-функций (port-Hamiltonian / Dirac structure + Ring-LWE шум).

## 2. MWC128 — базовая конструкция и PRNG-hash

MWC128 использует 128-битное состояние и работает по схеме:

$x ← (A · x) mod P$, где $P = A · 2^{64} − 1$ — большое простое число (числа Ризеля, но могут применяться и числа Прота).

Типичный хэш-вариант использования MWC-итерации в составе Хэш-функции (из PRNG_hash.md):

```c
uint64_t mwc128_hash(const uint8_t* data, uint64_t len, uint128_t seed) {
    uint128_t h = unmix(seed);
    for (int i = 0; i < len >> 3; i++) {
        h += *(uint64_t*)data; data += 8;
        h = (uint64_t)h * A1 + (h >> 64);  // MWC-итерация
    }
    // обработка остатка + финальный mix
    return mix(h);
}
```

**Выбор множителя A1** — ключевой: A1 должен быть генератором мультипликативной группы в ℤ/2⁶⁴ℤ, обеспечивать хороший avalanche и равномерность заполнения - это дополнительные требования.

**Миксеры** (MXM, MUM, MWC, RRX, RNG, MRC):
- MXM — классика Murmur/SplitMix (xor-shift + multiply)
- MUM — multiply → xor/add reduction
- RRX — rotate-xor (циркулянтные матрицы над GF(2))
- MRC — CRT-подход с двумя модулями

**Векторизация и folding** — позволяют параллельный хэш на AVX-512 / NEON, lazy reduction, со слиянием хеш-сегментов.

**Тестирование** — SMHasher (avalanche, BIC, collisions, speed), TestU01, custom SAC / difficulty. MWC128 проходит все 188 тестов SMHasher3.

#### 3. Обратимые MWC-генераторы (Reversible RNG)

Обратимость достигается через *Shoup-модульное умножение* на константу (без переполнения) и использование A как обратного значения к сдвигу $2^s$.

Пример MWC32 reverse (из ReversibleRNG.md):

```c
uint32_t MWC32_rev(uint32_t s, uint32_t A) {
    const uint32_t p = (A << 16) - 1u;
    const uint64_t w = 0x1p48 / p;          // предвычисленный \bar{w}
    uint64_t q = (s * w) >> 32;
    uint64_t r = ((s - q * A) << 16) + q;
    return (r < p) ? r : r - p;
}
```

Аналогично для MWC64/128 — обратное умножение на $A$ (или деление на $2^s$).

**Аналогии**:
- LCG: x_{k-1} = a^{-1} (x_k - c) mod p
- PCG: ядро обратимо, scrambler должен быть биективным
- XorShift: обратная матрица (инверсия сдвигов и обратимая бинарная матрица трансформации)

**Применение**:
- adjoint Monte-Carlo / обратные симуляции
- undo/redo в играх и симуляциях
- stochastic rounding в низкоточностных нейросетях
- постквантовые схемы (комбинация с Ring-LWE шумом)

## 4. NTT-friendly дизайн и связь с числами Прота

**Выбор модуля** — $p = k · 2^s + 1$ (Proth prime) с примитивным корнем порядка $2^s$ → позволяет использовать полный NTT размером $2^s$.

**Связь с циклотомическими полиномами** $Φ_{2^{s+1}}(X) ≈ X^{2^s} + 1$ → диагонализация циклических сдвигов и умножений на X в NTT-базисе.

**Гибридный раунд**:
- MWC-step (линейный carry + multiply)
- NTT-mix (быстрое полиномиальное умножение / NTT в кольце)
- Аффинные (обратимые) преобразования двоичных векторов на входе и выходе

**Преимущества**:
- O(n log n) на микс вместо O(n²)
- jump/skip через возведение в степень в NTT-домене
- естественная интеграция с hardware-ускорителями NTT (FHE, lattice crypto)

## 5. Конструкция sponge / PRNG-хэш на обратимом MWC + NTT

- **Состояние**: 256–1024+ бит (несколько 128-битных MWC-слов)
- **Absorb**: MWC-step + добавление данных + NTT-mix
- **Squeeze**: несколько раундов чистого миксера (без ввода)
- **Обратимость** используется для: предсказания орбит, merge сегментов, undo операций, adjoint-вычислений
- **Нелинейность** (опционально): квантовый шум, простые ARX-слои, нелинейная H(x) в Dirac-стиле

## 6. Оценка качества и слабые места

**Lattice test / спектральный тест** — индикатор качества линейных генераторов (MWC, LCG, XorShift).

Параметры спектрального теста LCG:
- d_s — минимальное расстояние между гиперплоскостями в s-мерном пространстве
- F_s = d_s · m^{1/s} (чем выше — тем лучше, идеал > 0.8–1.2)

Проблема LCG: низкие F_s в малых размерностях → точки лежат на регулярной решётке → предсказуемость. Это не проявляется явно для MWC (когдруэнтный генератор по модулю простого числа) при стпени 64 и выше. Генераторы проходят статистические тесты TestU01 включая Furier, LineaComp и Спектральные тесты Кнута. Однако, проявляется периодическая структура следования паттернов в тесте "Trump Aces" и наличие двух непересекающихся орбит, одна из которых исключает степени двойки "тузы".

**Рекомендации по усилению**:
- Увеличение состояния и числа раундов
- Weyl-increments / добавление квантового шума (Ring-LWE стиль)
- Нелинейные слои (модульное сложение, rotate-xor и переставноки байт)
- Dirac-структура, как индуцированный базис (кососимметричная J + диссипация R)

## 7. Перспективы: физическая интерпретация и Ring-LWE

Стохастическая модель на базе *Dirac structure + NTT-диагонализация матрицы перестановок+ квантовый шум*:

- J → вращение мод в NTT-базисе (консервативная часть)
- R → диссипация / затухание
- ξ ~ Discrete Gaussian в ℤ_q → Ring-LWE безопасность

Это позволяет:
- O(n log n) шаг генерации
- спектральный анализ для атаки (если шум мал)
- градиентное обучение параметров (J, R, σ) для модели с выбранной структурой

Связь с FHE (CKKS/BFV) — прямая: NTT как ключевая оптимизация, шум как основа безопасности. Построение векторного хеш с использованием CRT-MRC. RAG поиск по сжатым данным. Доказательства с нулевым разглашением.
